# AGENTS.md (docs)

## 適用範囲

- このファイルの指示は `C:\Users\jacks\Documents\flowlet\docs` 配下に適用する。
- ルートの `AGENTS.md` と矛盾する場合は、より制約が強い方を優先する。

## 対象

- プロジェクト文書全般 (`*.md`)

## 基本方針

- 実装と矛盾しない内容を優先し、推測で仕様を断定しない。
- 手順は再現可能なコマンド単位で記述する。
- 1変更1目的を守り、文書構成の大規模改変は分離する。

## 記述方針

- 見出し構造は既存スタイルを維持する。
- コマンド・パス・設定キーはバッククォートで明示する。
- 必要に応じて「前提」「手順」「検証」「既知の制約」を分けて書く。

## 検証方針

- 追加・更新した手順は、可能な範囲で実行確認する。
- 実行できない場合は、未検証であることと前提条件を明記する。

## Skills運用

- 正本テンプレート: `docs/SKILLS_POLICY_TEMPLATE.md`
- 更新時はテンプレートを先に更新し、レイヤー固有文言以外を同期する。
- skillは「明示指定された場合」または「タスク内容がskill説明に明確に一致する場合」にのみ使う。
- 複数候補がある場合は、目的達成に必要な最小セットだけを使う。
- skill適用時は、使うskill名と適用理由を1行で先に共有する。
- skill適用時も当該方針を優先する（実装と矛盾しない記述を維持し、推測で仕様を断定しない）。
- skill実行時も既存の安全方針を厳守する（権限昇格は目的を1文で添えて承認を求める）。
- skill実行後は当該ファイルの検証方針に従って確認する。
- 検証不可の場合は、理由・代替確認・残リスクを明記する。
